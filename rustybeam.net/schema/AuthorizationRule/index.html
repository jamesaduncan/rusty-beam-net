AuthorizationRule Schema

The AuthorizationRule schema defines access control policies for HTTP requests based on path patterns, HTTP methods, user roles, usernames, and CSS selectors.

Schema Definition

Schema URL: https://rustybeam.net/schema/AuthorizationRule

Properties

Property Type Cardinality Description
path Text 1 Path pattern to match (supports wildcards and {username} placeholder, e.g., "/admin/*", "/users/{username}/*")
method Text 1 HTTP method(s) to match (supports wildcards and comma-separated values, e.g., "GET", "PUT,POST,DELETE", "*")
role Text 0..1 Required user role for access (e.g., "administrators", "users")
username Text 0..1 Required username for access (supports {username} placeholder for matching authenticated user)
selector Text 0..1 CSS selector to restrict access to specific HTML elements (e.g., "ul.entry")
action Text 1 Action to take when rule matches ("allow" or "deny")

Examples
